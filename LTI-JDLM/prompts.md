# 📌 Prompts utilizados

## 1. Descripción del software
```
Asuma el rol de Product Manager y Arquitecto de Software.
Diseñe un ATS (Applicant Tracking System) innovador para la startup LTI. 
Incluya una breve descripción del software, su valor añadido y ventajas competitivas.
```

## 2. Funciones principales
```
Explica las funciones principales de un ATS moderno con enfoque en eficiencia, colaboración en tiempo real, automatización y asistencia de IA.
```

## 3. Lean Canvas
```
Genera un Lean Canvas para un ATS con IA, colaboración en tiempo real y automatización avanzada. 
Incluye problema, clientes, propuesta de valor, solución, canales, ingresos, costes, métricas clave y ventaja injusta.
```

## 4. Casos de uso
```
Describe 3 casos de uso principales de un ATS moderno (ejemplo: registrar candidato automáticamente, colaboración en feedback, automatización de entrevistas). 
Incluye un diagrama sencillo para cada caso.
```

## 5. Modelo de datos
```
Diseña el modelo de datos de un ATS. 
Incluye entidades, atributos (nombre y tipo) y relaciones entre ellas.
```

## 6. Diseño de alto nivel
```
Explica y dibuja el diseño de alto nivel de un ATS moderno. 
Incluye frontend, backend API, base de datos, servicios de IA e integraciones externas.
```

## 7. Diagrama C4
```
Genera un diagrama C4 para el módulo de gestión de candidatos de un ATS. 
Incluye contexto, contenedor, componentes y relaciones.
```
